# Hi-Voicer 模型说明

## 录音和转录不是同一个模型

录音不需要 AI 模型，它只是把麦克风声音采集成音频。

真正把声音变成文字的是 ASR 模型，也就是转录模型。Hi-Voicer 后续的语音输入和文件转录会共用同一类 ASR 模型，只是使用场景不同：

- 语音输入：短音频、实时性优先
- 文件转录：长音频、准确率和时间戳优先

## 当前模型列表策略

当前列表分两类：

1. 可自动安装  
   这类模型是直接 zip 包，软件可以下载、解压、放到固定目录并保存配置。

2. 需后续接入引擎  
   这类模型本身更强，但不是下载一个 zip 就能直接用，还需要对应推理引擎。例如 Qwen3-ASR、Whisper、FunASR/SenseVoice。

## 当前预设

| 模型 | 状态 | 用途 |
| --- | --- | --- |
| Vosk 中文轻量 | 可自动安装 | 先跑通第一版，低配置电脑也能用 |
| Vosk 中文高精度 | 可自动安装 | 准确率更好，体积较大 |
| Qwen3-ASR 0.6B | 需后续接入引擎 | 后续中文实时识别优先候选 |
| Qwen3-ASR 1.7B | 需后续接入引擎 | 后续高精度候选 |
| OpenAI Whisper Base | 需后续接入引擎 | 多语言文件转录候选 |
| FunASR SenseVoiceSmall | 需后续接入引擎 | 中文增强候选 |

## 为什么第一版还保留 Vosk

Vosk 的优势是工程接入简单，模型下载形态稳定，适合第一版先打通“下载模型、配置模型、离线识别”的完整链路。

Qwen3-ASR、Whisper、SenseVoice 更适合做后续质量升级，但它们需要额外推理引擎、音频预处理和模型管理，不应该在没有接入前假装可用。
